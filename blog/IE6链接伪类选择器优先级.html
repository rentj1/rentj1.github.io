IE6链接伪类选择器优先级bug

代码: css /*reset*/ a:hover { text-decoration:underline; } a { text-decoration:none; } /*page*/ .test a{ text-decoration:none; } #test a{ text-decoration:none; } html: <a href="#">a</a> <div class="test"> <a href="#">.test a</a> </div> <div id="test"> <a href="#">#test a</a> </div>

说明：

1、a:hover 和 a

a

这里选择器“a:hover”使用了伪类优先级高于选择器“a”，之前以为后面a { text-decoration:none; }的会覆盖前面 a:hover {text-decoration:underline;}

2、a:hover 和 .test a

.test a

在IE6中选择器a:hover的优先级高于选择器“.test a”，鼠标移到链接上时链接有线，
在其他浏览器中选择器“.test a”的优先级高于选择器“a:hover”，鼠标移到链接上时链接没有线

3、a:hover 和 #test a

#test a

这里选择器“#test a”的优先级高于选择器“a:hover”，鼠标移到链接上时链接没有线
